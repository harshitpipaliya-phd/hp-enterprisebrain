# FEP-001 — Recommendations

## Recommendation Layer (linkage, not generation)

Recommendations are **generated by EPIC-006 (Recommendation Engine)**. The dashboard **links** each gap to its candidate recommendation/ESO.

- **Linkage rule:** a gap of root-cause family F links to an ESO whose `trigger.gapTypes` (Block 2) includes F. This is the only linkage rule — it reuses the contract, no new logic.
- **Composition hint:** for multi-family gaps, the dashboard may show the `lineage.composedOf` (Block 12) chain of an ESO (e.g., Leadership = Diagnose ∘ Prioritize ∘ Decide ∘ Communicate).
- **Trust gating:** a recommended ESO is only actionable if the operator's effective autonomy (F-001.5 / Block 5 `trustLevels` ceiling) permits it. The dashboard greys out ESOs above the ceiling and routes them to EPIC-007 human approval.
- **Open Case link:** if a `Case`/`Hypothesis` already exists for the gap, the dashboard prefers linking to it over spawning a new one.
- **Ranking:** recommendations are ordered by gap severity × ESO match confidence; the dashboard displays rank, it does not compute the ESO selection (EPIC-006 does).
